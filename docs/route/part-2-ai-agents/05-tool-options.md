# AIツールの選択肢を知る

## この章でできるようになること

コーディングに使うAIツールの代表的な種類を知り、どのような違いがあるのかを大まかに説明できるようになります。

第0部では、CodexまたはClaude Codeのどちらかを使える状態にしました。
この章では、それが唯一の選択肢ではないことも確認します。

## まず知っておくこと

AIツールは、いくつかのタイプに分けて見ると理解しやすくなります。

- Webチャット
- ターミナルで使うコーディングエージェント
- エディタ統合型の補助ツール
- GitHub上のレビューや補完ツール
- ローカルLLMと連携するツール

どれが常に最適という話ではありません。
作業内容、環境、費用、入力する情報、チームの方針によって向き不向きが変わります。

## ターミナルで使うコーディングエージェント

CodexやClaude Codeのようなツールは、ターミナルから使うコーディングエージェントとして扱えます。

特徴は次です。

- 作業ディレクトリの文脈を扱いやすい
- ファイルを読んで説明できる
- 変更案を出せる
- 許可された範囲で編集やコマンド実行を行える
- Git差分やビルド結果と組み合わせやすい

この教材で早めに導入したのは、学習中に「今見ている教材」と「AIへの質問」をつなげたかったからです。

## エディタ統合型のAIツール

GitHub CopilotやCursor系のツールのように、エディタの中で補完や編集支援を行うものもあります。

特徴は次です。

- 開いているファイルの補完が得意
- 書きながら提案を受けやすい
- UI上で差分を確認できる場合がある
- エディタの使い方に慣れていると便利

一方で、この教材ではVS Codeなど特定のエディタを必須にしません。
ターミナルとブラウザを中心に、環境差を少なくして進めます。

## GitHub上で使うAI支援

GitHub上にも、コードレビューやPull Request作成を助けるAI支援があります。

これは第7部や第8部で、GitHubを使う段階になってから理解しやすくなります。
今の段階では、「ローカルでAIエージェントを使うこと」と「GitHub上で支援を受けること」は別だと知っておけば十分です。

## ローカルLLM連携型ツール

ローカルLLMと連携するツールもあります。

手元のPCやサーバーでモデルを動かせるため、入力情報の扱いを自分で管理しやすい場合があります。
ただし、セットアップ、モデル選択、PC性能、速度、品質の確認が必要です。

この教材の最初のルートでは、ローカルLLMを必須にしません。
後の学習パスとして扱うほうが、初学者には進めやすいからです。

## やってみる

AIに、用途別にツールを整理してもらいます。

```text
開発学習で使うAIツールを、次の用途ごとに整理してください。

- 用語を質問する
- ローカルのリポジトリを読ませる
- 小さなファイル変更を頼む
- エディタでコード補完を受ける
- GitHub上のPull Requestを確認する
- ローカルLLMを試す

この教材では、まずCodexまたはClaude Codeを使う前提です。
他の選択肢は、今すぐ乗り換えるためではなく、位置づけを理解するために説明してください。
まだファイルは変更しないでください。
```

## 何が起きたのか

第0部でCodexまたはClaude Codeを選んだのは、教材リポジトリを見ながら学ぶ体験を早く作るためです。

他のツールを知らなくてよい、という意味ではありません。
ただし、最初からすべてを比較し始めると、学習の入口が重くなります。

まずは1つのエージェントを使いながら、モデル、ツール、作業ディレクトリ、変更確認の考え方を身につけます。

## 運用者の視点

AIツールを選ぶときは、機能だけでなく運用も見ます。

- 入力内容がどこに送られるか
- ログが残るか
- チームや組織で使ってよいか
- 料金が発生するか
- ファイル変更を確認しやすいか
- 誤操作したときに戻せるか

個人学習でも、この視点を少しずつ持っておくと安全です。

## AIに聞いてみよう

```text
私は今、CodexまたはClaude Codeを使って学習しています。
他のAIコーディング支援ツールにはどんな種類があるか、
今すぐ乗り換える前提ではなく、位置づけがわかるように整理してください。

比較するときは、モデル、ツール、エディタ連携、GitHub連携、ローカルLLMを分けてください。
まだファイルは変更しないでください。
```
