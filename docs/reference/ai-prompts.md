# AIへの聞き方

このページでは、Claude CodeやCodexに依頼するときのプロンプト例をまとめます。

## 基本方針

AIには、いきなり「やって」ではなく、次の順番で依頼します。

```text
理解して
↓
計画して
↓
説明して
↓
必要なら変更して
↓
確認して
```

## リポジトリを読ませる

```text
このリポジトリのREADME.mdとdocs/route/part-0-start/01-concept.mdを読み、
この教材の目的、対象読者、最終成果物を要約してください。
まだファイルは変更しないでください。
```

## コマンドの意味を聞く

```text
次のコマンドが何をするのか、初心者向けに1行ずつ説明してください。
また、実行前に注意すべきことがあれば教えてください。

<ここにコマンドを貼る>
```

## SSH設定を確認する

```text
GitHubへSSHで接続する準備をしています。
公開鍵と秘密鍵の違い、ssh-keygen、ssh-add、ssh -T git@github.com の意味を、
初心者向けに説明してください。
秘密鍵を公開してはいけない理由も教えてください。
```

## エラーを相談する

```text
次のエラーについて、原因候補を整理してください。
私は初心者なので、まず何を確認すべきかを順番に教えてください。

環境：
実行したコマンド：
エラー：
期待していた結果：
```

## 変更前に計画を出させる

```text
次の目的でファイルを変更したいです。

目的：

変更する前に、変更対象ファイル、変更理由、確認方法を説明してください。
まだファイルは変更しないでください。
```

## 公開前レビュー

```text
このリポジトリをGitHub Pagesで公開する前にレビューしてください。

確認観点：
- 秘密情報や個人情報が含まれていないか
- READMEが最低限の説明になっているか
- 公開して問題がある業務情報が含まれていないか
- AIが生成した内容に不自然な点がないか
- 公開後に修正する手順がわかるか

問題点があれば、重要度順に指摘してください。
まだファイルは変更しないでください。
```
