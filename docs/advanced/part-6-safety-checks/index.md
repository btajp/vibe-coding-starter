# 発展編 第6部：安全装置と確認手順を作る

第6部では、AIの出力を信じ切るのではなく、差分、build、test、lint、リンクチェックで安全に受け止める手順を作ります。

AIに任せる範囲が広がるほど、確認手順を先に用意することが重要になります。

## この部のゴール

AIの変更を受け入れる前に、何を確認すべきかを説明し、自分のプロジェクト用のチェックリストを作れるようになることがゴールです。

## この部で学ぶこと

- `git status` と `git diff` を確認の入口にする
- build、test、lint、リンクチェックの役割
- 危険コマンドと状態を変える操作の確認
- 秘密情報、`.env`、トークン、APIキーの確認
- 公開前チェックリスト
- 失敗した変更を立て直す考え方
- AIに確認コマンドを実行させる前の注意

## 予定する章立て

### 1. 差分確認を入口にする

AIが何を変えたかを、`git status` と `git diff` で確認します。

### 2. build、test、lintを分ける

それぞれが何を確認するためのものかを整理します。

### 3. 秘密情報を確認する

`.env`、トークン、APIキー、秘密鍵が差分や公開対象に入っていないかを確認します。

### 4. 公開前チェックリストを作る

GitHub Pagesや公開リポジトリに出す前の確認項目をまとめます。

### 5. 失敗した変更を立て直す

AIの変更が崩れたとき、何を止め、何を確認し、どこからやり直すかを扱います。

### 6. 第6部の確認

自分のプロジェクト用のcommit前・公開前チェックリストを作ります。

## 到達目標

- AIの変更を受け入れる前に、確認すべき項目を説明できる
- 自分のプロジェクト用のチェックリストを作れる
- 危険な変更を、実行前または公開前に止められる
