# 発展編 第10部：自分専用のAI開発環境を作る

第10部では、自分の成果物リポジトリに、AIと継続開発するための作業環境を整えます。

発展編で学んだAGENTS.md、要件メモ、プロンプトテンプレート、skills、安全確認、AIレビュー、サブエージェント運用を、自分のプロジェクトに合わせてまとめます。

## この部のゴール

自分のプロジェクトに、AIと継続開発するための基本セットを導入し、今後も育てられる状態にすることがゴールです。

## この部で学ぶこと

- 自分のプロジェクト用AGENTS.mdを書く
- よく使うプロンプトテンプレートをまとめる
- 必要ならskillsを作る
- build、test、review、commit前確認の流れを整える
- AIに任せる範囲と、人間が確認する範囲を決める
- 今後の開発で育てていくルールを残す

## 予定する章立て

### 1. 自分のプロジェクトを選ぶ

AIと継続開発したい成果物リポジトリを選びます。

### 2. AGENTS.mdを作る

プロジェクトの目的、編集方針、安全方針、commit前確認を整理します。

### 3. プロンプトテンプレートを置く

よく使う依頼を、READMEやdocs配下のメモとしてまとめます。

### 4. 必要ならskillsを作る

繰り返し使う専門手順がある場合だけ、skill化を検討します。

### 5. 確認手順を整える

build、test、review、commit前確認を、自分のプロジェクトで実行できる形にします。

### 6. 継続的に育てる

AIとの作業で見つかった改善点を、AGENTS.md、テンプレート、チェックリストへ反映します。

### 7. 発展編の確認

自分のプロジェクトで、AIに何を任せ、何を人間が確認するかを説明します。

## 到達目標

- 自分のプロジェクトに、AIと継続開発するための作業環境を導入できる
- AIと継続開発するための基本セットを説明できる
- AIに任せる量を増やしても、安全確認の軸を失わない
