# 発展編 第2部：AGENTS.mdを育てる

第2部では、リポジトリごとの作業方針をAGENTS.mdに書き、AIの振る舞いを安定させます。

AGENTS.mdは、一度書いたら終わりの設定ファイルではありません。
AIとの作業で見つかった失敗、好み、禁止事項、確認手順を少しずつ反映して育てていく対象として扱います。

## この部のゴール

自分のプロジェクトに合ったAGENTS.mdを書き、AIに更新案を出させ、変更前後でAIの出力がどう変わるかを確認できるようになることがゴールです。

## この部で学ぶこと

- AGENTS.mdは何を書く場所か
- 教材本文と、AIエージェント向け作業方針を分ける理由
- 文章の粒度、画像方針、問題方針、安全方針の書き方
- AIにやってほしくないことを禁止事項として書く方法
- AIにAGENTS.mdの改善案を出させる方法
- AIにAGENTS.md自体を編集させるときの確認手順
- 作業中に見つかったルールをAGENTS.mdへ反映する方法
- 同じ依頼でも、AGENTS.mdの有無や内容で出力が変わることを確認する実験
- AGENTS.mdを変更したあと、AIが新しい指示を読めているか確認する方法
- AGENTS.mdに書きすぎると読みにくくなる問題
- CLAUDE.mdやAGENTS.mdが肥大化したときの分割方針
- 全体方針、プロジェクト固有ルール、再利用する手順を分ける考え方

## 予定する章立て

### 1. AGENTS.mdの役割を確認する

AGENTS.mdが教材本文ではなく、AIエージェント向けの作業方針であることを確認します。

### 2. 最小のAGENTS.mdを書く

日本語で書く、勝手にcommitしない、危険な操作は事前に説明するなど、最小限の方針を書きます。

### 3. 同じ依頼をAGENTS.md変更前後で比べる

同じ依頼を出し、AGENTS.mdの内容によって出力がどう変わるかを体験します。

### 4. AIにAGENTS.mdの改善案を出させる

うまくいかなかったAIの振る舞いから、AGENTS.mdに追加すべきルールをAIと一緒に考えます。

### 5. AGENTS.mdの肥大化を防ぐ

全体方針、プロジェクト固有ルール、再利用する手順、リファレンス、skillsの置き場所を分けます。

### 6. 第2部の確認

自分のプロジェクトで、AGENTS.mdに残すものと外へ逃がすものを整理します。

## 到達目標

- 自分のプロジェクトに合ったAGENTS.mdを書ける
- AIの失敗から、次に守るべきルールを抽出できる
- AGENTS.mdの変更前後で、AIの出力差を比較できる
- AGENTS.mdをAIに更新させる前に、変更予定と理由を確認できる
- AGENTS.mdに残すもの、リファレンスやskillsへ逃がすものを判断できる
