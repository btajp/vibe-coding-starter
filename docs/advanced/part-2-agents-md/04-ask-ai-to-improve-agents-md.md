# AIにAGENTS.mdの改善案を出させる

この章では、AIとの作業で見つかった違和感を、AGENTS.mdの改善案に変える方法を扱います。

AIが期待と違う動きをしたとき、毎回その場で注意するだけでは、同じことが繰り返されます。
繰り返したくないことは、AGENTS.mdのルール候補に変えていきます。

## この章でできるようになること

- AIの失敗や違和感から、AGENTS.mdに追加するルール候補を作れる
- 事実、原因の推測、追加ルールを分けて整理できる
- AIにAGENTS.mdを編集させる前に、変更案を確認できる

## まず事実を分ける

AIに改善案を出させる前に、何が起きたのかを短く整理します。

ここで大切なのは、いきなり「AIが悪い」と決めつけないことです。
依頼が曖昧だったのか、AGENTS.mdに書いていなかったのか、AIが見落としたのかを分けて考えます。

整理する観点は、次の3つです。

- 何を頼んだか
- AIが何をしたか
- 何が期待と違ったか

## ルール候補に変える

たとえば、AIがファイルを編集する前に変更予定を説明しなかったとします。

その場合、次のように整理できます。

| 観点 | 内容 |
| --- | --- |
| 何を頼んだか | 章本文を修正してほしい |
| AIがしたこと | すぐにファイルを編集した |
| 期待と違ったこと | 編集前に変更予定を説明してほしかった |
| 追加するルール候補 | ファイル編集前に、変更予定ファイルと理由を説明する |

このように、出来事をそのままルールにするのではなく、次回も使える形に変えます。

## ルールは短く書く

AGENTS.mdに追加するルールは、短く、実行できる形にします。

よい例です。

```markdown
- ファイル編集前に、変更予定のファイルと理由を説明してください。
```

少し弱い例です。

```markdown
- いい感じに安全に作業してください。
```

「いい感じに」は、人によって意味が変わります。
AIにも判断しづらい言葉です。

AGENTS.mdでは、AIにしてほしい行動がわかる形にします。

## AIに改善案を出させる

AIにAGENTS.mdの改善案を出させる場合は、すぐ編集させません。
まず、追加候補だけを出してもらいます。

```text
今のAIの作業で、期待と違う点がありました。

起きたこと:
- 依頼: 章本文を少し修正してほしい
- AIの動き: 変更予定を説明せず、すぐファイルを編集した
- 期待: 編集前に変更予定ファイルと理由を説明してほしかった

この出来事をもとに、AGENTS.mdへ追加するルール候補を3つ以内で提案してください。

条件:
- まだAGENTS.mdは編集しない
- ルールは短く、次回も使える形にする
- 似た内容が既にAGENTS.mdにある場合は、追記ではなく既存ルールの修正案にする
```

この依頼では、AIに「編集」ではなく「改善案の抽出」をさせています。
人間が採用するか判断してから、AGENTS.mdを更新します。

## 編集させるときの確認

改善案を採用する場合も、AIにいきなり編集させません。
先に、変更予定を出してもらいます。

```text
提案のうち、1つ目のルールをAGENTS.mdに反映したいです。

編集する前に、次を説明してください。

- 変更する場所
- 追加または修正する文面
- 既存ルールと重複しないか
- 変更後に確認するコマンド

私が了承するまで、まだファイルは編集しないでください。
```

この手順を挟むと、AGENTS.mdが場当たり的に増えるのを防ぎやすくなります。

## やってみる

過去のAIとの作業を1つ思い出し、次を埋めます。

```text
依頼:

AIの動き:

期待と違ったこと:

次回から守ってほしい行動:

AGENTS.mdに追加するなら一文で:
```

実際にAGENTS.mdへ追加するかは、あとで判断します。
まずは、出来事をルール候補に変える練習です。

## AIに聞いてみよう

AIに、出来事を一問一答で整理してもらうこともできます。

```text
AIとの作業で起きた違和感を、AGENTS.mdの改善案に変えたいです。

次の条件で、私に質問してください。

- 質問は5問
- 一問一答形式にする
- 1問ずつ質問し、私の回答を待つ
- 各質問では、A/B/Cの選択肢も毎回表示する
- A/B/Cだけで答えにくい場合は、短く自由記述してよいことも書く
- 5問が終わったら、AGENTS.mdへ追加するルール候補を3つ以内で出す
- まだファイル編集、削除、commit、pushはしない

目的は、AIの失敗を責めることではなく、次回の作業方針に変えることです。
```

AIに質問してもらうと、何が問題だったのかを落ち着いて分解できます。

## 何が起きたのか

AGENTS.mdは、AIの失敗や人間の好みを、次回以降の作業方針に変えるための場所です。

ただし、何でも追加すればよいわけではありません。
一度きりの出来事なのか、繰り返し避けたいことなのかを見極めます。

次章では、AGENTS.mdに書きすぎて読みにくくなる問題を扱います。
