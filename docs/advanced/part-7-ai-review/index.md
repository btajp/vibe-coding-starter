# 発展編 第7部：AIレビューを設計する

第7部では、AIに実装させるだけでなく、複数の観点でレビューさせる流れを作ります。

レビューは「AIの答えを信じる」ためではなく、人間が判断する材料を増やすために使います。

## この部のゴール

AIにレビューしてほしい観点を分けて依頼し、レビューコメントを対応するものと見送るものに分けられるようになることがゴールです。

## この部で学ぶこと

- 実装レビュー
- 初学者視点レビュー
- セキュリティレビュー
- 文章レビュー
- 画像と本文の対応レビュー
- 差分レビュー
- レビュー観点をAIに明示する方法
- レビュー結果をそのまま信じず、人間が採用判断する方法
- 実装AIとレビューAIを分ける意味

## 予定する章立て

### 1. [レビュー観点を分ける](01-review-perspectives.md)

安全、文章、動作、初学者視点など、レビューの目的を分けます。

### 2. [実装レビューを頼む](02-implementation-review.md)

AIが変更したコードや文書を、差分ベースで確認させます。

### 3. [初学者視点レビューを頼む](03-beginner-review.md)

説明不足、順序の違和感、画像が必要な箇所を確認させます。

### 4. [セキュリティレビューを頼む](04-security-review.md)

秘密情報、危険なコマンド、公開前確認の観点から見てもらいます。

### 5. [レビュー結果を採用判断する](05-adopt-review-results.md)

AIの指摘をそのまま反映せず、対応するものと見送るものに分けます。

### 6. [第7部の確認](06-review-ai-review.md)

同じ差分に対して、複数の観点でレビューを依頼し、結果を比較します。

## 到達目標

- AIにレビューしてほしい観点を分けて依頼できる
- レビューコメントを、対応するものと見送るものに分けられる
- 実装AIとレビューAIを分ける意味を説明できる
