# 発展編 第9部：長期タスクをAIに任せる

第9部では、AGENTS.md、要件メモ、プロンプト、skills、安全確認、レビュー設計を使い、長めの開発タスクを小さな確認単位で進めます。

ここまでに作った作業環境を前提にして、ようやくAIに大きめの作業を任せます。

## この部のゴール

長期タスクをAIに頼む前に、条件、確認方法、チェックポイントを整理し、途中で方針がずれたときに止めて立て直せるようになることがゴールです。

## この部で学ぶこと

- AIと壁打ちして要件を作る
- AIに質問してもらい、回答を要件メモにまとめる
- 目的、制約、完了条件、やらないことを分ける
- 再帰型ToDoリストを作る
- チェックポイントを置く
- 調査、設計、実装、レビュー、検証を分ける
- 長期タスクが崩れたときに立て直す
- AIに任せた結果を人間が説明する

## 予定する章立て

### 1. 長期タスクを任せる前に

AIに任せる目的、制約、完了条件、やらないことを整理します。

### 2. AIに質問してもらう

AIに質問役を頼み、人間が回答する形で要件を引き出します。

### 3. 要件メモから計画を作る

要件メモを読み、調査、設計、実装、レビュー、検証に分けます。

### 4. 再帰型ToDoリストを作る

各章や各機能ごとに、次のToDoを展開できる形にします。

### 5. チェックポイントで止める

一定の作業ごとに差分、build、レビューを確認します。

### 6. 崩れた長期タスクを立て直す

方針がずれたときに、どこまで戻るか、何を読み直すかを整理します。

### 7. 第9部の確認

小さな長期タスクを1つ選び、要件メモ、ToDo、チェックポイントを作ります。

## 到達目標

- 長期タスクをAIに頼む前に、条件と確認方法を整理できる
- 大きな変更を小さな作業単位に分けられる
- 途中で方針がずれたときに、止めて立て直せる
